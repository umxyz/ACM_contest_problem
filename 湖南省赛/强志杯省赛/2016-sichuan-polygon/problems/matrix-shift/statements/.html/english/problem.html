Matrix Transformation
time limit per test
1 second
memory limit per test
1024 megabytes
input
standard input
output
standard output

Bobo has a matrix of n rows and n columns. The rows are numbered by 0, 1, ..., (n - 1) from top to bottom, and the columns are numbered by 0, 1, ..., (n - 1) from left to right. The cell in the intersection of the i-th row and the j-th column is denoted as (i, j). For each cell (i, j), there is a number i × n + j written in.

Bobo is going to perform q transformations successively. The transformations are of 2 kinds. The i-th transformation is of ti-th kind, and it's described by 3 parameters li, ri, di.

If ti = 1, the number in cell where li ≤ x ≤ ri, 0 ≤ y < n will be transferred to the cell (x, y) by the transformation.

If ti = 2, the number in cell where 0 ≤ x < n, li ≤ y ≤ ri will be transferred to the cell (x, y) by the transformation.

Note that means the remainder of a after division by b.

Bobo would like to know the final configuration of the matrix.

Input

The first line contains 2 integers n, q (1 ≤ n ≤ 200, 1 ≤ q ≤ 105).

The i-th of the following q lines contains 4 integers ti, li, ri, di ().

Output

n lines. The i-th line contains n integers ai, 0, ai, 1, ..., ai, n - 1 which denotes the final number in cell (i, j).

Examples

Input
3 2
1 1 1 1
2 1 1 1
Output
0 5 2
4 7 3
6 1 8
Input
3 1
1 0 2 1
Output
1 2 0
4 5 3
7 8 6
