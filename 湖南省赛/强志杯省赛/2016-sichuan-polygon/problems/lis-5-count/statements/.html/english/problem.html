Bobo is tired of all kinds of hard LIS (Longest Increasing Subsequence) problems, so he decides to make himself some easier one.

Bobo wants to build a sequence of integers {x1, x2, ..., xn}, where xi lies in the range [ai, bi] (that is, ai ≤ xi ≤ bi).

Let be the length of longest increasing subsequence of {x1, x2, ..., xn}. It's clear that . Bobo would like to find gk which is the number of sequences whose for k = 1, 2, ..., n.

Note that a sequence {i1, i2, ..., ik} is a increasing sequence of {a1, a2, ..., an} only if:

  • 1 ≤ i1 < i2 < ... < ik ≤ n
  • ai1 < ai2 < ... < aik

Input

The first line contains an integer n (1 ≤ n ≤ 5).

The i-th of the following n lines contains 2 integers ai, bi (1 ≤ ai ≤ bi ≤ 103).

Output

n integers g1, g2, ..., gn.

Examples

Input
2
1 2
1 2
Output
3 1
